# WorldBlox Spec — C-295 Phase 1 Bootstrap

WorldBlox is the HIVE world-state grammar for translating Mobius integrity signals into a playable civic world.

This is not a separate source of truth. WorldBlox is a projection layer inside `mobius-hive`.

## Core placement

- `mobius-hive` owns the WorldBlox JSON contract and world projection artifacts.
- `mobius-browser-shell` renders WorldBlox as a read-only surface and sends player actions through `/api/hive/action`.
- `Civic-Protocol-Core` may later attest meaningful world events, but Browser Shell must not canonize them directly.
- `Mobius-Substrate` remains canon / protocol / meaning.
- The ledger remains the durable truth layer.

## Non-negotiable truth flow

Canon → Ledger → UI

WorldBlox must preserve this rule:

- UI may render blocks.
- UI may request actions.
- UI may keep local save state.
- UI may not invent canon.
- UI may not mint MIC.
- UI may not seal world truth.
- HIVE may simulate pressure and meaning, but human merge is required for canon.

## Block model

A WorldBlox block is a small, verifiable world object. It can represent terrain, civic structures, sentinel anchors, events, quests, or overlays.

Every block should answer five questions:

1. What is this object?
2. Where does it belong?
3. What integrity state does it reflect?
4. What player or agent actions can interact with it?
5. What proof/canon boundary prevents it from becoming fake truth?

## Initial block types

| Type | Meaning | Example |
| --- | --- | --- |
| `zone` | Spatial civic area | Castle, Market, Aquifer, Archive |
| `structure` | Civic building or gameplay object | Beacon, Fountain, Vault Gate |
| `event_overlay` | Temporary world pressure | Signal Fog, Fountain Murmur |
| `quest_anchor` | Player interaction loop | Prepare the Seal |
| `sentinel_anchor` | Agent projection point | ZEUS Gate, JADE Mirror, HERMES Relay |
| `memory_node` | OAA/world memory projection | Reflection Stone |
| `ledger_marker` | Read-only proof marker | Attested pulse, sealed event pointer |

## Required semantic states

WorldBlox must use the shared Mobius semantic definitions:

- `attested` = quorum-approved
- `sealed` = cryptographically finalized
- `quarantined` = failed or unsafe
- `timed_out` = quorum not reached in time
- `legacy` = pre-canonical data

Do not redefine these terms for game convenience.

## Block lifecycle

1. `draft` — proposed world object; safe for docs and fixtures.
2. `simulated` — generated by HIVE world projection.
3. `attested` — quorum-approved by upstream proof/canon process.
4. `sealed` — cryptographically finalized by ledger path.
5. `quarantined` — failed, unsafe, or invalid object.
6. `legacy` — pre-canonical or historical projection retained for continuity.

## MVP contract

C-295 Phase 1 only adds the spec and schemas. It does not change runtime behavior.

Next phases may add:

- `world/blocks/*.json`
- `world/worldblox-state.json`
- `schemas/worldblox.schema.json` validation in CI
- Browser Shell WorldBlox renderer
- `/api/hive/action` expansion for block interactions

## Guardrails

WorldBlox must never collapse layer boundaries:

- Vault ≠ Canon ≠ Ledger
- UI ≠ truth
- Replay ≠ mutation
- Historical attestation ≠ real-time attestation
- HIVE projection ≠ ledger finality
