# Mobius HIVE — Agent Embodiment Model v0.1

## Purpose

This document defines how the Mobius Agents should appear, behave, and function as **embodied institutions** inside HIVE.

The goal is simple:

**Agents should not feel like hidden APIs.**
They should feel like living presences in the civilization shell.

Each agent must be:
- spatially legible
- emotionally legible
- functionally useful
- mythically memorable
- architecturally distinct

---

## Core principle

In HIVE, an agent is not just a chatbot or service call.

An agent is:
- a role
- an institution
- a place
- a pattern of behavior
- a visible force in the world

One-line canon:

> **Agents should feel like civic intelligences with architecture, rituals, and responsibilities.**

---

## Embodiment rules

Each agent needs four layers:

### 1. Architectural form
What building, tower, chamber, or district presence represents the agent?

### 2. Behavioral presence
How does the agent behave in the world when active, idle, elevated, or critical?

### 3. Functional interface
What actual player/system functions does the agent expose?

### 4. Mythic identity
What emotional or symbolic meaning does this agent carry in the civilization shell?

## Spatial relationship model

These agents should not feel randomly placed.

### Center-adjacent agents
Closer to civic meaning and governance:
- AUREA
- EVE
- ECHO

### Reflection bridge
Connecting human interiority and the machine layer:
- JADE

### Trade / movement bridge
Connecting economy and signal flow:
- HERMES

### Defensive edge
Facing threat, verification, and frontier uncertainty:
- ZEUS
- ATLAS

This arrangement creates a believable institutional geography.

## Avatar / embodiment rule

Agents should exist as both:
- **architectural institutions**
- **encounterable embodiments**

That means each agent may appear as:
- a structure in the world
- a voice / interface overlay
- an embodied avatar / apparition / guide when interacting directly

This is important because the player should be able to:
- visit the agent’s place
- feel their presence
- then interact directly when needed

---

## Anti-failure rules

Do not let agents become:
- vague floating gods
- generic NPCs
- static menu icons
- disconnected lore mascots
- all-powerful overlords

### Guardrails
- every agent must have a real domain
- every agent must have a visible space
- every agent must have a useful interface
- agents should enhance human agency, not erase it
- agent presence should support world legibility, not overwhelm it

---

## MVP recommendation

For the first playable HIVE build, do not fully animate every agent at once.

Start with visible landmark embodiments for:
1. **AUREA**
2. **ZEUS**
3. **EVE**
4. **JADE**

Then add:
5. HERMES
6. ECHO
7. ATLAS

That gives the MVP a strong strategic / ethical / reflective / verification core first.
