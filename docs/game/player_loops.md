# Mobius HIVE — Player Loops v0.1

## Core thesis

HIVE should feel like a living civic world, not just a menu-driven app.

The player loop must connect:
- exploration
- learning
- contribution
- governance
- reflection
- economy

---

## Primary player loops

## 1. Spawn → Explore → Discover
Player enters Citizen Plaza, sees active districts, discovers landmarks, portals, quests, and civic signals.

**Purpose:** onboarding and emotional orientation

**Outputs:**
- unlocked districts
- discovered NPCs / agents / institutions
- first tasks and social hooks

---

## 2. Learn → Practice → Earn
Player enters School of AI / OAA, takes a lesson, challenge, or apprenticeship quest, then receives XP, token utility, or civic reputation.

**Purpose:** growth and useful participation

**Outputs:**
- progression
- skill unlocks
- educational proof
- visible world contribution pathways

---

## 3. Contribute → Improve World → Leave Mark
Player repairs, builds, teaches, verifies, or supports a civic process. The world changes in response.

**Purpose:** make contribution visible and meaningful

**Outputs:**
- repaired districts
- archive entries
- new structures, lights, banners, or landmarks
- trust / integrity progression

---

## 4. Reflect → Archive → Re-enter
Player journals in Reflection Gardens, closes a cycle, or processes a major event. Reflection creates memory and strengthens continuity.

**Purpose:** make inner life part of the civilization loop

**Outputs:**
- private journal artifacts
- public memory traces when appropriate
- morale recovery
- continuity anchors

---

## 5. Propose → Vote → World Changes
Player participates in governance: proposals, oath staking, elder legitimacy, civic votes, seasonal rituals.

**Purpose:** make governance embodied and consequential

**Outputs:**
- changed world state
- altered district rules
- new public policies
- elder seat shifts and legitimacy effects

---

## 6. Trade → Stake → Route Value
Player uses Market Quarter to exchange resources, stake for governance, burn for ritual, or route value into systems.

**Purpose:** spatialize the economy

**Outputs:**
- utility spending
- governance weight
- symbolic scarcity
- social and institutional value flow

---

## Daily return loop

```text
Login
→ spawn in Citizen Plaza
→ read world state / cycle / active events
→ choose one activity:
   - learn
   - trade
   - reflect
   - govern
   - build / repair
→ complete task
→ visible world response
→ archive / close loop
→ return tomorrow
```

This loop should be short enough for casual return, but deep enough for long-term identity formation.

---

## Role-based loop emphasis

### Scout
- explore
- gather
- discover
- complete field quests

### Citizen
- participate
- trade
- learn
- reflect
- vote

### Agent
- route signals
- observe integrity
- support systems
- coordinate special functions

### Elder
- govern
- defend legitimacy
- steward districts
- survive public scrutiny

---

## World design rule

Each loop should produce one of four outcomes:
- **progress**
- **memory**
- **trust**
- **world change**

If a loop produces none of those, it will feel hollow.

---

## Canon sentence

**The player should never feel like they are grinding a menu. They should feel like they are inhabiting and shaping a civilization.**
