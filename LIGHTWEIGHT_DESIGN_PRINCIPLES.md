# Mobius HIVE — Lightweight Design Principles v0.1

## Purpose

This document defines how Mobius HIVE and the Mobius Substrate stay deep, expressive, and durable without becoming bloated or heavy.

One-line canon:

> **Depth should come from interaction, not bulk.**

---

## Core thesis

HIVE should feel rich because:
- rules interlock
- districts respond
- agents have clear domains
- governance creates visible consequence
- memory preserves meaning

It should not depend primarily on:
- giant asset libraries
- duplicated systems
- oversized menus
- ornamental lore weight
- content sprawl without stronger civic logic

---

## Principle 1 — System-dense, not asset-dense

Prefer:
- reusable rules
- templates
- state machines
- modular environment pieces

over:
- huge bespoke content volume
- oversized cinematic payloads
- one-off mechanics that do not generalize

### Rule
If a small rule system can create the same lived depth as a much larger content payload, prefer the rules.

---

## Principle 2 — Substrate first, spectacle second

### The Substrate holds
- identity
- memory
- agent roles
- governance rules
- integrity logic
- event definitions

### HIVE renders
- district state
- architecture
- landmarks
- transitions
- public consequence

### Rule
Do not store spectacle in the substrate.
Do not rebuild constitutional logic inside the world layer.

---

## Principle 3 — Reuse one interaction grammar

Prefer one compact loop:

```text
inspect → open overlay → act → world responds → memory updates
```

Use this across:
- Reflection
- Market
- Archive
- Governance
- Building identity
- White Room command flow

### Rule
Add new systems through existing grammar before inventing new interaction languages.

---

## Principle 4 — Make layers behavioral, not file-heavy

A district can feel different through state changes instead of many separate handcrafted versions.

### Core district states
- nominal
- elevated
- critical
- festival / renewal

### Response channels
- lighting
- banners
- NPC density
- route openness
- agent activity
- repair or scar visibility

### Rule
Prefer fewer spaces with stronger state response over many static spaces.

---

## Principle 5 — Clear domains reduce weight

### Agent clarity
- AUREA = strategy
- ZEUS = verification
- EVE = ethics / synthesis
- JADE = reflection / continuity
- HERMES = routing / exchange
- ECHO = pulse / continuity relay
- ATLAS = watch / anomaly scan

### District clarity
- Castle = authority
- Archive = memory
- Market = exchange
- Reflection = inner life
- School = growth
- Agent Citadel = institutional intelligence

### Rule
If two agents or districts are doing the same thing without a strong reason, the architecture is getting heavy.

---

## Principle 6 — Modular art beats brute-force art volume

Prefer:
- stylized world design
- iconic silhouettes
- landmark-based recognition
- modular pieces
- symbolic state changes
- expressive lighting

### Rule
Use art to clarify the civilization, not to bury it under file weight.

---

## Principle 7 — Memory should be selective, durable, and legible

Preserve:
- cycle anchors
- important oaths
- key proposals
- major challenges
- founding canon
- significant reflections
- structural civic events

Avoid preserving raw noise without compression.

### Rule
Store what shapes continuity, not every possible event artifact.

---

## Principle 8 — One strong Dome before many weak worlds

Start with:
- one Primary Dome
- five strong districts
- one White Room
- one onboarding path
- one oath/proposal/challenge chain
- one building progression system

Then expand.

### Rule
A strong small world is more convincing than a large but blurry universe.

---

## Principle 9 — Build engines, not disconnected features

### Core engines
- Identity Engine
- Memory Engine
- Governance Engine
- District State Engine
- Agent Role Engine
- Transit Engine
- Building Progression Engine

### Rule
Prefer a system that can generate many meaningful scenarios over many features that each only do one thing.

---

## Principle 10 — Every feature must strengthen a civilizational primitive

A feature should strengthen one or more of:
- identity
- memory
- contribution
- governance
- continuity
- world legibility
- social belonging

### Rule
If it strengthens none of those, it is probably ornamental weight.

---

## Principle 11 — World-first meaning, overlay-second precision

The world should orient the player emotionally.
The overlay should help the player act precisely.

### Good pattern
- walk to place
- inspect place
- open anchored overlay
- act
- see world consequence

### Rule
Every important overlay should feel anchored to a place in the civilization.

---

## Principle 12 — Private depth, public clarity

### Public layer
- districts
- rituals
- landmarks
- visible governance outcomes

### Inner layer
- White Room
- council synthesis
- continuity review
- strategic command structures

### Rule
Do not force the player to understand the entire substrate before they can meaningfully inhabit the world.

---

## Principle 13 — Multi-agent systems must feel institutional

Agents should be treated as:
- institutions
- domains
- architectural presences
- recurring intelligences with bounded responsibility

### Rule
Add agents only when they represent a stable constitutional need, not a temporary task convenience.

---

## Principle 14 — Ritual should add gravity, not friction theater

Good ritual is:
- visible
- memorable
- consequential
- tied to legitimacy and memory

### Rule
Ritual is justified when it increases weight, witness, and meaning.

---

## Principle 15 — Optimize for returnability

A player should be able to:
- do something meaningful in three minutes
- feel more grounded in ten minutes
- go deep when desired

### Rule
The civilization should welcome repeated entry without demanding full cognitive reboot every session.

---

## MVP recommendation

Prioritize:
- one Primary Dome
- five strong districts
- one White Room
- one onboarding flow
- one building progression ladder
- one oath / proposal / challenge chain
- one compact district-state response system

---

## Canon sentences

**Depth should come from interaction, not bulk.**

**Make the substrate small and truthful. Make the world expressive and reusable. Let richness emerge from consequence rather than weight.**
