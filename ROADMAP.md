# Mobius HIVE — Roadmap

## Purpose

This roadmap turns the `mobius-hive` repository from a doctrine archive into a staged build program.

It defines:
- what HIVE is trying to become
- what gets built first
- what gets deferred
- how the civilization shell grows from concept to playable world

One-line canon:

> **Doctrine gives the world meaning. The roadmap gives the world sequence.**

---

## Strategic goal

Build **Mobius HIVE** as a ledger-native civilization shell where:
- identity becomes architecture
- contribution becomes visible world change
- governance becomes embodied ritual
- memory becomes persistent infrastructure
- agents become institutions inside the world

The first objective is **not** a giant MMO.
The first objective is a **coherent Primary Dome** that proves the shell works.

---

## Build philosophy

### Build order
1. **Doctrine first**
2. **Spatial shell second**
3. **Playable loops third**
4. **Social/governance depth fourth**
5. **Expansion and multi-biome civilization fifth**

### Why
If HIVE is built as pure content first, it will drift into a generic game.
If it is built as a civilization shell first, then each mechanic has meaning.

## Phase 1 — Primary Dome Blockout
### Goal
Create the first playable spatial shell of HIVE.

### Scope
- one Dome
- five priority districts
- movement shell
- landmark visibility
- basic inspect system
- basic map / routing

### MVP districts
1. Citizen Plaza
2. Castle of Elders
3. Archive Vault
4. Market Quarter
5. Reflection Gardens

### Build outputs
- environment blockout
- district landmark models/placeholders
- spawn logic
- minimap prototype
- world-state banner

### Success condition
A player can enter the Dome, orient immediately, travel between core districts, and understand the civilization shell.

---

## Phase 2 — World-to-Overlay Action Layer
### Goal
Connect place to function.

### Scope
- reflection overlay
- market overlay
- archive overlay
- governance overlay
- building/profile overlay

### Rule
Every major place must map to a real action.

### Build outputs
- inspect prompts
- contextual overlays
- world-preserving modal/panel system
- action completion feedback

### Success condition
The player can move through the world and complete meaningful actions without the world feeling fake or irrelevant.

---

## Phase 3 — Identity and Contribution Layer
### Goal
Make player presence visible and persistent.

### Scope
- player structures / chambers
- 4-tier building progression MVP
- visible contribution markers
- reflection traces
- role-based spatial identity

### Build outputs
- personal dwelling/chamber
- progression triggers
- visible building states
- contribution-based architectural changes

### Success condition
A player’s structure begins to tell the story of what they have done for the world.

---

## Phase 4 — Civic Economy Layer
### Goal
Make value flow visible and meaningful inside the world.

### Scope
- basic market flow
- staking / burn / lock surfaces
- civic reward logic
- public-goods contribution pathways

### Build outputs
- market UI and space coupling
- first resource sinks and sources
- visible district economic state shifts
- governance-linked value actions

### Success condition
The economy feels civic and embodied, not like a detached wallet mini-app.

---

## Phase 5 — Governance Layer
### Goal
Make the Castle feel alive.

### Scope
- seven visible thrones
- legitimacy indicators
- support / oppose oath actions
- one active challenge path
- archive of current/former holders

### Build outputs
- Elder seat shell
- public governance board
- oath interface
- legitimacy state visualization
- challenge staging flow

### Success condition
Authority becomes visible, scarce, and challengeable.

---

## Phase 6 — Agent Institution Layer
### Goal
Embodied agents become active world institutions.

### Scope
- AUREA
- ZEUS
- EVE
- JADE
first, then:
- HERMES
- ECHO
- ATLAS

### Build outputs
- agent landmark embodiments
- basic direct interactions
- multi-agent event visuals
- role-specific functional overlays

### Success condition
Agents feel like places and intelligences inside the Dome, not hidden service calls.

---

## Phase 7 — Social Life and Public Presence
### Goal
Make the Dome feel inhabited by more than one person.

### Scope
- visible avatars/presence
- shared events
- district activity density
- social discovery
- public ritual participation

### Build outputs
- ambient social layer
- public presence indicators
- event congregation logic
- lightweight social interaction shell

### Success condition
The player feels they are inhabiting a civilization with others, not a solo world with menus.

---

## Phase 8 — Seasonal Civilization Loop
### Goal
Turn HIVE into a repeating but evolving civic cycle.

### Scope
- 90-day Elder seasons
- seasonal transitions
- district state changes
- archive permanence + political turnover
- festival timing

### Build outputs
- seasonal reset logic where appropriate
- public transition rituals
- archive carry-forward systems
- throne turnover flow

### Success condition
HIVE starts behaving like a living civilization instead of a static world.

---

## Phase 9 — Secondary Biome Prototype
### Goal
Prove that HIVE can scale beyond one Dome without losing identity.

### Recommended first choices
- **Desert Dome** for scarcity/trade pressure
- **Orbital Dome** for advanced strategic civilization framing

### Build outputs
- one secondary biome spec translated into blockout
- inter-dome travel prototype
- archive continuity check
- shared economy continuity check

### Success condition
A second world feels like a new pressure on the same civilization, not a disconnected map.

---

## Phase 10 — HIVE Civilization Network
### Goal
Expand from one world to a true multi-dome civilization.

### Scope
- multiple biomes
- shared governance layers
- inter-dome transit
- cultural skins / local flavor
- one civilizational archive and economic backbone

### Success condition
HIVE becomes a civilization network rather than a single hub world.

---

## What to avoid

Do not let the roadmap drift into:
- content sprawl before shell coherence
- cosmetics before institutional logic
- monetization before civic meaning
- dozens of zones before one strong Dome
- vague AI flavor before agent embodiment rules

### Guardrail
If a feature does not strengthen:
- place
- memory
- contribution
- governance
- identity
- continuity

then it is probably not first-order HIVE work.
